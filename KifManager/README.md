# KifManager

棋譜のダウンロードと抽出を扱う補助ツールを置くフォルダです。

## scripts/

`scripts/` には、floodgate、WCSC、電竜戦、その他の棋譜から USI `position` 形式の棋譜行を抽出するCLIスクリプトと、棋譜ダウンロード用スクリプトを置いています。

## kif-manager.py

`kif-manager.py` は、棋譜抽出と棋譜ダウンロードを扱う Tkinter GUI です。

## 必要なもの

Python 3 と `cshogi` が必要です。`.7z` アーカイブを入力フォルダに置いて抽出する場合は `py7zr` も必要です。WCSC16以前の `.lzh` アーカイブを扱う場合は `lhafile` も必要です。

Windowsでは、Python Launcher の `py` を使うのが一般的です。

```bat
py -m pip install cshogi py7zr lhafile
```

Linux/macOSでは、環境に合わせて `python3` などを使ってください。

```bash
python3 -m pip install cshogi py7zr lhafile
```

## 起動

Windows:

```bat
py kif-manager.py
```

Linux/macOS:

```bash
python3 kif-manager.py
```

BookMiner から呼び出す場合は次のオプションを使います。

Windows:

```bat
py kif-manager.py --from_bookminer
```

Linux/macOS:

```bash
python3 kif-manager.py --from_bookminer
```

この場合、棋譜抽出の出力ファイルは BookMiner が読む `BookMiner/book/think_sfens.txt` に強制設定されます。

上位タブは `棋譜抽出` と `棋譜のダウンロード` に分かれています。

`棋譜抽出` には、floodgate、WCSC、電竜戦、その他の抽出タブがあります。各タブで入力フォルダと出力ファイルを指定し、必要ならプレイヤー名フィルタを指定して `抽出` を押します。出力ファイルは `think_sfens.txt` がデフォルトです。floodgate の入力フォルダは `downloaded-kif/floodgate`、WCSC の入力フォルダは `downloaded-kif/wcsc` がデフォルトです。

floodgate では `開始日` / `終了日` を指定して、2026/06/01 から 2026/06/07 までのように対局日で絞り込めます。`2026/1/1` や `2026-1-1` のように月日を1桁で指定しても構いません。開始日に `2024` のように年だけを指定すると 2024/01/01、終了日に年だけを指定すると 2024/12/31 として扱います。

floodgate の `min-rating` は、指定期間内に一度でもそのrating以上になったプレイヤー同士の棋譜を抽出します。`負けた棋譜` にratingを指定すると、指定期間内に一度でもそのrating以上になったプレイヤーが負けた棋譜も追加します。`引き分けた棋譜` にratingを指定すると、指定期間内に一度でもそのrating以上になったプレイヤーが引き分けた棋譜も追加します。相手のratingは問いません。GUIでは `負けた棋譜` と `引き分けた棋譜` のデフォルトは4000です。空欄にするとそれぞれのrating条件は無効です。

WCSC では `開始年` / `終了年` を指定して、2020年から2025年までのような範囲抽出ができます。WCSCの `wcso1` は WCSC30 扱いで、2020年として扱います。

WCSCと電竜戦では `決勝に出場したソフトの棋譜のみ抽出` を有効にできます。WCSCでは同じ大会の決勝棋譜から、電竜戦では同じ本戦イベントの決勝リーグまたはA級棋譜からプレイヤー名を集め、そのどちらかが登場する予選/二次予選/決勝棋譜だけを抽出します。

`抽出対象のファイルをログに出力` を有効にすると、抽出対象として選択された棋譜ファイルのパスをログに出力します。デフォルトはオフです。

`棋譜のダウンロード` には、floodgate、WCSC、電竜戦のダウンロードタブがあります。

ダウンロード中は `ダウンロード` ボタンが `停止` に変わります。押すと停止要求を出し、現在処理中の1ファイルまたは1ページが終わったところで停止します。

GUI のログ欄は、表示行数が増えすぎないように古い行を自動的に削除します。画面上には直近約1000行が残ります。

floodgateタブでは `2026` のように対局年を指定します。ダウンロードした年別アーカイブは、デフォルトでは `downloaded-kif/floodgate/wdoor2026.7z` のように保存します。今年のものは前日までの棋譜がダウンロードされますが、保存名は日付を付けず `wdoorYYYY.7z` に統一します。サーバー上のファイルサイズと既存ファイルのサイズが同じ場合は、ダウンロードを省略します。サイズが異なる場合は `.tmp` にダウンロードしてから `wdoorYYYY.7z` に置き換えます。`前日分もダウンロードする` と `当日分もダウンロードする` はデフォルトでオンです。前日分は年別アーカイブ更新時刻の都合で漏れやすい前日5:00以降の棋譜を補うため、floodgate の日別ページから取得します。当日分は floodgate の today ページから取得します。どちらも出力フォルダ配下の `YYYYMMDD/` に `.csa` ファイルを保存します。

WCSCタブでは `WCSC36` のように大会名を指定します。デフォルトでは `downloaded-kif/wcsc/wcsc36/` のように、出力フォルダ配下の大会名フォルダへ保存します。`live中継ページからダウンロード` はデフォルトでオフです。オフの場合は `http://www2.computer-shogi.org/kifu/kifu.html` の公式棋譜アーカイブから `.zip` または `.lzh` をダウンロードして展開します。ZIP/7zアーカイブはサーバー上のファイルサイズを確認し、前回正常に展開したアーカイブと同じサイズなら再ダウンロードを省略します。ダウンロードが必要な場合は `.tmp` に保存し、サイズ確認後に使用します。展開時も、既存の棋譜ファイルとアーカイブ内のファイルサイズが同じなら書き換えず、サイズが異なる場合だけ置き換えます。オンにした場合だけ、WCSC24以前の `live2` 形式などのlive中継ページから個別棋譜を取得します。WCSC16以前のLZH展開には `lhafile` が必要です。`kifu6.lzh` のように `lhafile` だけでは展開できない古い圧縮形式は、7-Zip、bsdtar、unar のいずれかを外部コマンドとして使用します。Windowsでは、7-Zip が標準のインストール先に入っていれば `C:\Program Files\7-Zip\7z.exe` または `C:\Program Files (x86)\7-Zip\7z.exe` を自動検出します。PATH に `7z` を追加していなくても構いません。WCSC30は中止になりましたが、オンライン大会として開催された棋譜は `WCSO1` と指定すれば `downloaded-kif/wcsc/wcso1/` にダウンロードできます。サーバー負荷を避けるため、アクセス間隔のデフォルトは 10 秒で、通常は 2 秒以上を指定します。

### Windowsで古いWCSC棋譜を展開する場合

WCSC16以前の公式棋譜アーカイブは `.lzh` 形式です。通常は Python ライブラリ `lhafile` で展開しますが、古い圧縮方式の一部は `lhafile` だけでは展開できません。

Windowsでは 7-Zip をインストールしておくと、KifManager が必要に応じて 7-Zip の `7z.exe` を使って展開します。標準のインストール先である次のどちらかに `7z.exe` があれば自動検出します。

```text
C:\Program Files\7-Zip\7z.exe
C:\Program Files (x86)\7-Zip\7z.exe
```

PATH に `7z` を追加している場合も使用されます。

電竜戦タブでは、公式リンク集に掲載されている大会をコンボボックスから選択できます。`一覧更新` を押すと、公式リンク集から最新の大会一覧を取得します。デフォルトでは `downloaded-kif/denryu/dr6_production/` のように、出力フォルダ配下の大会キーごとのフォルダへ保存します。`live中継ページからダウンロード` はデフォルトでオフです。オフの場合は大会ページまたは公式リンク集から一括ZIP/7zを探して展開します。ZIP/7zアーカイブはサーバー上のファイルサイズを確認し、前回正常に展開したアーカイブと同じサイズなら再ダウンロードを省略します。サイズが異なる場合は `.tmp` にダウンロードし、展開時も同サイズの既存棋譜ファイルは書き換えません。一括ZIP/7zが見つからない大会では、live中継ページの `kifulist.txt` から `kifufiles/*.csa` を取得する方式にフォールバックします。大会終了前で一括ZIP/7zがまだ公開されていない場合も、このlive取得を使います。アクセス間隔のデフォルトは 10 秒で、通常は 2 秒以上を指定します。

出力は 1 対局 1 行です。

```text
startpos moves 7g7f 3c3d ...
```

`その他` の抽出では、平手以外の初期局面も `sfen ... moves ...` 行として出力できます。

```text
sfen <initial-sfen> moves 7g7f 3c3d ...
```

同一の position 行は初出だけを出力します。

### プレイヤー名フィルタ

`both-player-list` は、先手と後手の両方がリスト内の正規表現に一致する棋譜だけを通します。

`either-player-list` は、先手または後手の少なくとも片方がリスト内の正規表現に一致する棋譜だけを通します。

両方を指定した場合は OR 条件です。どちらかの条件を満たす棋譜が抽出されます。

### 駒落ちフィルタ

`その他` の抽出タブでは、`駒落ちの棋譜を除く` がデフォルトで有効です。初期局面が平手ではなく、かつ盤上と持駒の合計が40枚未満の棋譜を駒落ちとして除外します。

### 設定保存

GUI の入力値は `kif-manager-settings.pickle` に保存され、次回起動時に復元されます。この設定ファイルはローカル設定なので git 管理対象外です。
